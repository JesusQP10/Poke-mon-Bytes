PLAN PARTE 1 - POKEMON ORO 

Fecha de referencia: 2026-02-13

==================================================
1) OBJETIVO DE ESTA PARTE
==================================================
Construir la primera seccion jugable desde:
- PRESS START en la intro (Ho-Oh)
hasta:
- llegar a la casa del Senor Pokemon (Route 30)

==================================================
2) ALCANCE FUNCIONAL
==================================================
Fase de flujo canon de inicio:
1. Title Screen (Ho-Oh) + PRESS START
2. Menu principal (NUEVA PARTIDA / CARGA o CONTINUE si existe save / OPCIONES)
3. Si se elige NUEVA PARTIDA: Intro de Oak
4. Seleccion de hora (clock)
5. Spawn en habitacion del jugador
6. Conversacion con mama
7. Ir al laboratorio de Elm
8. Dialogo de Elm + eleccion de starter
9. Salir de New Bark
10. Ruta hacia Cherrygrove
11. Cherrygrove (minimo funcional para pasar)
12. Route 30 hacia casa de Mr. Pokemon
13. Llegada a la casa del Senor Pokemon

Opcional (si hay tiempo):
- Conversacion completa dentro de la casa (Mr Pokemon + Oak + Pokedex + Egg).


==================================================
3) ALCANCE TECNICO
==================================================
Motor base en Phaser:
- SceneManager (Title, Intro, Overworld, UIOverlay)
- AssetLoader por escena
- Sistema de input (A/B/Start/Select + d-pad)
- Escalado pixel-perfect

Overworld:
- Movimiento 4 direcciones por tile
- Colisiones
- Warps (puertas, escaleras, transiciones entre mapas)
- Camera follow + limites del mapa
- Interaccion con tiles y NPCs (tecla A)

Sistema de eventos:
- Script engine por comandos simples (showText, moveNPC, setFlag, ifFlag, warp, giveItem)
- Flags persistentes por partida

UI:
- Text box estilo GBC
- Menu de opcion (SI/NO)
- Dialogos con paginacion

Combate minimo para Parte 1:
- Trigger de encuentros salvajes en hierba
- Transicion a batalla
- Resultado de batalla y retorno al overworld

Persistencia:
- Save/Load con estado minimo (mapa, posicion, direccion, flags, reloj, starter)

8) BACKEND NECESARIO PARA PARTE 1
==================================================
APIs minimas:
- GET /game/state
- POST /game/save
- POST /game/flags
- POST /game/starter

Modelo Save minimo:
- userId
- mapId
- x, y, direction
- currentScene
- clock (hour, minute, day)
- starter
- party
- inventory basico
- flags (json)
- updatedAt

- El frontend ejecuta el runtime.
- El backend persiste estado y valida acciones sensibles.


==================================================
9) CRITERIOS DE TERMINADO 
==================================================
Parte 1 esta completada cuando:
- Se puede empezar en title y pulsar start
- Aparece menu principal y funciona Nueva Partida / Carga-Continue / Opciones
- Intro Oak se reproduce sin glitches
- Se configura hora
- Jugador puede moverse en mapa inicial
- Evento con mama funciona
- Evento con Elm + starter funciona
- Se llega caminando hasta casa Mr Pokemon
- Guardar/cargar mantiene posicion y progreso
- No hay bloqueos criticos en el flujo canon
